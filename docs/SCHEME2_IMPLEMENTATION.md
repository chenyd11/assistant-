# 方案2实现指南：视频库 + 改写版本库（子表）

## 📊 数据库结构设计

### 主表：视频库（已存在）
管理原始视频和转录内容

### 子表：改写版本库（需创建）
管理所有改写后的内容版本

---

## 🛠️ 步骤1：在飞书Bitable中创建子表

### 操作步骤：

1. **打开你的Bitable**
   https://fcnrncf2hy2l.feishu.cn/base/V87Cb06erar7kGsTFm0cR0JZnof

2. **创建子表**
   - 点击左侧「+」按钮
   - 选择「新建数据表」
   - 命名为：**改写版本库**

3. **添加字段**（按以下顺序）：

| 字段名称 | 字段类型 | 说明 | 示例值 |
|---------|---------|------|--------|
| 版本编号 | 文本 | 唯一编号 | VER_2026_0001_01 |
| 关联视频编号 | 文本 | 关联主表视频 | VID_2026_0001_76d70ac1 |
| 平台 | 单选 | 发布平台 | 公众号/小红书/抖音 |
| 账号 | 文本 | 具体账号名 | 我的公众号/个人生活号 |
| 版本标题 | 文本 | 改写后的标题 | 从"黑中介"手中死里逃生... |
| 内容 | 长文本 | 完整改写内容 | （文章内容） |
| 字数 | 数字 | 内容字数 | 1500 |
| 标签 | 多选 | 内容标签 | 干货/避坑/职场 |
| 创建时间 | 日期 | 创建日期 | 2026-02-15 |
| 发布时间 | 日期 | 发布日期（可选） | 2026-02-16 |
| 发布链接 | 超链接 | 发布后链接 | https://mp.weixin.qq.com/... |
| 状态 | 单选 | 当前状态 | 草稿/审核中/已发布/已删除 |
| 备注 | 文本 | 其他说明 | 待优化开头... |

4. **配置单选选项**：
   
   **平台选项：**
   - 公众号
   - 小红书
   - 抖音
   
   **状态选项：**
   - 草稿（灰色）
   - 审核中（黄色）
   - 已发布（绿色）
   - 已删除（红色）

---

## 🔄 步骤2：关联关系说明

### 如何关联：

**主表（视频库）的一条记录：**
```
视频编号: VID_2026_0001_76d70ac1
标题: 进厂打工避坑指南
```

**子表（改写版本库）的多条记录：**
```
记录1: VER_0001_01 | VID_2026_0001_76d70ac1 | 公众号 | 我的公众号 | ...
记录2: VER_0001_02 | VID_2026_0001_76d70ac1 | 小红书 | 个人生活号 | ...
记录3: VER_0001_03 | VID_2026_0001_76d70ac1 | 小红书 | 职场干货号 | ...
记录4: VER_0001_04 | VID_2026_0001_76d70ac1 | 抖音 | 主账号 | ...
记录5: VER_0001_05 | VID_2026_0001_76d70ac1 | 抖音 | 小号 | ...
```

通过「关联视频编号」字段，可以：
- 在主表查看该视频的所有改写版本
- 在子表筛选某个视频的所有版本
- 统计每个视频产生了多少个版本

## 🎯 完整工作流程（方案2）

```
1. 视频入库
   ↓
2. 自动转录 → 保存到主表
   ↓
3. 洗稿改写 → 生成3平台版本
   ↓
4. 自动保存到子表（改写版本库）
   ↓
5. 在子表中编辑/审核每个版本
   ↓
6. 发布 → 更新子表状态为"已发布"
   ↓
7. 填写发布链接
```
