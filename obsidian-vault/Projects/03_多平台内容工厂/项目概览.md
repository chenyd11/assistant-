# 🏭 多平台内容工厂

## 🎯 项目目标

用 OpenClaw 跑通：给定内容 → 一键批量洗稿 → 生成适合多平台的稿件 → 自动发布。

## 🔄 工作流程

```
输入（视频/文章/选题）
    ↓
OpenClaw 处理
    ├─→ 洗稿（视频转文字→改写→文风调整）
    ├─→ 生成图片（AI绘图→可编辑格式）
    ├─→ 三平台格式适配
    │       ├─→ 公众号版（深度长文）
    │       ├─→ 小红书版（emoji+配图）
    │       └─→ 抖音图文版（短平快）
    ↓
自动发布（多账号批量）/ 待审核发布
```

## 🚧 当前卡点（2026-02-17）

### 卡点1：洗稿文风需要调整
**问题**：当前文风不够灵活，需要更适配不同平台调性
**解决思路**：
- 设计「文风模板库」（小红书风/公众号风/抖音风）
- 基于平台特点调整 Prompt
- 参考 #1 AI小说 的理论，加入「负面约束」规则

### 卡点2：图文内容生成
**问题**：
- AI生成图片质量需要把控
- 需要「可修改的图文」（而非固定图片）

**解决思路**：
- 图片生成：Midjourney/SD API → 生成候选图 → 人工筛选
- 可编辑图文：
  - 方案A：生成 Canva/Figma 可编辑模板
  - 方案B：生成 PPT/Keynote 格式，人工调整
  - 方案C：Markdown + 图片占位符，导入编辑器

### 卡点3：多平台多账号批量发布
**问题**：技术方案不确定

**备选方案**：
| 方案 | 描述 | 优先级 |
|-----|------|--------|
| **A. 问原作者「字流」** | 了解其实现方案 | 🟢 先尝试 |
| **B. 第三方批量发布软件** | 扫码登录多账号 | 🟡 备选 |
| **C. 各平台官方API** | 最稳定但限制多 | 🟡 长期方案 |
| **D. 浏览器自动化** | Playwright/Selenium模拟 | 🔴 维护成本高 |

## 🛠️ 技术栈

- 视频处理：ffmpeg
- 语音转文字：Whisper
- 改写引擎：Kimi/GPT（需文风模板优化）
- 图片生成：Midjourney/Stable Diffusion API
- 图文编辑：Canva API / Figma API / PPT生成
- 发布：
  - 方案待定（字流/第三方/官方API/自动化）
- 调度：OpenClaw cron
